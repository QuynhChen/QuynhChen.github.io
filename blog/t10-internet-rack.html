What is a Rack on the Internet? 12/9/14
Dear Fellow Beginners,

Rack provides a minimal, modular and adaptable interface for developing web applications in Ruby by interfacing between webservers supporting Ruby(webservers that support Rack: Phusion Passenger, Litespeed, Mongrel, Thin, Ebb, Webrick) and your Ruby frameworks(your app).

http://rack.github.io/

To use Rack: write an "app" aka Ruby framework . An "app" is an object that responds to the #call method, taking the environment hash as a parameter, and returning an Array with three elements:

  • The HTTP response code ("status")
  • A Hash of headers
  • The response body, which must respond to #each

Two ways to provide this "app"

#1 Directly by touching a my_rack_app.rb file. In this rb file, type in:

require 'rack'

app = Proc.new do |env|
  ['200', {'Content-Type' => 'text/html'}, ['A barebones rack app.']]
end

Rack::Handler::WEBrick.run app

#2 use #rackup command line tool. Look for your config.ru file (it's probably located right below your Rakefile and it should require config/environment.rb). Type in the config.ru:

run Proc.new { |env| ['200', {'Content-Type' => 'text/html'}, ['get rack\'d']] }

Then on the command line in terminal, type in:

rackup config.ru

Rack does two things: 1) Racks specifies how a Rack app and the web server should communicate. A Rack app is an ruby object (not a class) that responds to #call. It takes exactly one argument, the environment (it's a hash!) and returns an Array of exactly three values: The status, the headers, and the body.
