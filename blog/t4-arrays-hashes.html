Array versus Hashes
Arrays arguments are reserved for Numbers, Hash arguments can be anything. 10/13/14
Dear Fellow Beginners,

What are Arrays and Hashes

Arrays are for a list of items in order. An array looks like this

a = [item1, item2].

Hashes are for keeping PAIRS of information, so that one item references the other item. A hash looks like this

a = {item1: item1, item2: item2}.

When to use array

Use arrays to (1) keep track of lists of strings, (2) store references to items you are working with, and (3)iterate through groups of objects in sequence.

The items are stored in Arrays in a numbered order. Where the first item is number 0, second item is number 1, the third is 2 and so on. By knowing their position in the array, you can write a function ask, what's the item in position number 4?

In addition, arrays can store any Ruby object such as integers, strings or floaters. so it could look like

a = [5, "bob", 8.2].

Lastly, to access the info stored in the array, the argument is always the number associated with the position. So if our list is a = [5, "bob", 8.2], then I can request for item 1 by writing a[1], which would give "bob." a[never not a number]

When to Use Hashes

Like Arrays, hashes also stores items but they store items in a different way. First, you use hashes to store pairs of items. Second, you cannot use Hash to put the items in order.

First, why do we want to store items in pairs? When we assign pairs, they become linked. So now we can ask for one, we get the linked info. The most common way to use hashes are menu. You know what you want to eat but don't know the pricing. We can ask for the food in the hash and we get back the price. So if the hash is:

a {"fries": 2, "burger":5, "love": "priceless"}

I can ask the matching number (assuming I plugged in the number to mean price) to burger.

a["burger"]

You will get 5.

Second, you can't use Hash to put the list in consecutive order. Unlike array's consecutive numbering scheme based on the item's position in the array, the hash is a bag of info. So if you want to know exactly where an item is, go use arrays.
