Dear Fellow Beginners,

Good question?

Bad questions are questions that do not clearly outline the specific problem. I've asked plenty of bad questions. For example, I've asked questions like: "I'm having trouble with ruby... can you fix it?" Or "How do I .length?" For the .length question, I think it could be improved by pointing the language, then the class that the method is to be worked on. And if possible, what my final goal was: "In Ruby, how do you use #length on an integer?"

Good questions are maps

Tell the person where you're at before you ask a question. "I'm looking at js, in DOM, I'm trying to create a function that does this but I get an error that says this!"

MY SOLUTION to "nums-to-words" Challenge

It felt So Good to play with this challenge this week that I'm showing off the answer in this blog too! (I'm sure it could be better but it tickles my soul as it is for right now.)

THE "nums-to-words" CHALLENGE: create a method that converts a number into the spelled out word so that in_word(10) would give "ten."

def in_words(num)
  ones = %w(zero one two three four five six seven eight nine)
  teens = %w(zero ten eleven twelve thirteen fourteen fifteen sixteen seventeen eighteen nineteen)
  tens = %w(zero ten twenty thirty forty fifty sixty seventy eighty ninety )

  if num.to_s.length == 1
    p ones.values_at(num).join
  elsif num.to_s.length == 2 && num < 20
    p teens.values_at(num.to_s.chars.map{|x| x.to_i}.first).join
  elsif num.to_s.chars.last == "0"
    p tens.values_at(num.to_s.chars.map{|x| x.to_i}.first).join
  else
    p (tens.values_at(num.to_s.chars.map{|x| x.to_i}.first) + ones.values_at(num.to_s.chars.map{|x| x.to_i}.last)).join(" ")
  end
end
