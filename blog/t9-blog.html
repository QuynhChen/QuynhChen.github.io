How to be a "Good" Programmer

Today, I asked a very good friend (ios guy) the ultimate question: How do you know when you have become a good programmer? The answer was not single worded but I extrapolated ONE WORD out of the conversation. The code would look "Beautiful."

Beautiful code means that an average programmer can easily read and understand the intent of the code with minimal effort.

So now, I should have THREE goals in my life. First, to be beautiful inside. Second, to be beautiful outside. Third, to create code that's beautiful too. Sounds like a journey already happening. Wish me luck!
