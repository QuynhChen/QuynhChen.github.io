Takeaway 1: NESTED Hashes

1. Create:

    cohort_hash = { raccoon:{quynh: {age:21}, Aarthi:{age:22} }, dog: {cutie:{height: shortie}, pie: {height: tall} } }

2. Access via key!

  1. cohort_hash[:key][:key][:key]
  2. cohort_hash.fetch(:raccoon).fetch(:quynh) #=> {age:21} or {:age => 21}
  3. cohort.fetch:quynh #same as cohort.fetch(:quynh)
    BONUS: Find key that doesn't exist
    1. cohort_hash[:feet] #=> nil
    2. cohort_hash.fetch(:feet){"no feet here"}
      #same as cohort_hash.fetch(:feet, "no feet here")
      #=>results "no feet here" instead of nil. Can add this to hash.

  1. Array.at(1).at(2)

3. Nested Array Problem:

    Input:
    given_array = [[:a, :b, :c], [:d, :e, :f], [:g, :h, :i], [:j, :k, :l], [:m, :n, :o], [:p, :q, :r], [:s, :t, :u], [:v, :w, :x]]
    Output:
    destination_array = an array with 3 nested arrays, each nested array contains 3 elements, each of these elements was a random selection from each nested array of the given array.
    Solution #1: Build the Goal
    Pseudocode:
    1. Create the goal array of an array with 3 nested arrays.
    2. Fill in each row and column with iteration.
    3. Isolate the sample to fill in.
    Solution:
      destination_array = Array.new(3){Array.new(3)}
      # creates [[nil, nil, nil], [nil, nil, nil], [nil, nil, nil]]

      destination_array.map!.with_index do |row, row_index|
        row.map!.with_index do |column, column_index|
          given_array.each_slice(3).to_a[row_index][column_index].sample
        end
      end

    Solution #2: Appreciate What's Given.
    Pseudocode:
    1. Change the given_array to be the destination array.
    2. Select each element from each nested array.
    Solution:

      destination_array = given_array.map(&:sample).each_slice(3).to_a

      #map iterates through each nested array and "sample" selects one element ("flattens the array").

      • #map(&;sample) is equivalent to map{|nested_array| nested_array.sample}

      #each_slice(3) groups each three elements.

      #to_a. Because each_slice(3) returns an enumerator method (I don't know WHAT that means), you need to call #t_a on the enumerator to see the results in array.

Heirarchy of organized data.

[least organized data] Abstract -> Data Type -> Data Structure [most organized]

Abstract Data Types (ADT): Just an idea of things and how to manipulate these things.
Boolean, the set of {true, false} and operations like negation, conditional, is equal to.
Integer, the set of {..., -2, -1, 0, 1, 2, ...} and operations like +, -, * and /.

Data Type: apply the ADT into real values.
Boolean, Integer, Fixnum, Symbol classes in Ruby examples: bool in C++, boolean in Java

Data Structure (DS) (unlike the above simple types, these are not "atomic" and can be divided into parts): apply ADT in an organized/structured relationship. Array, , hash, queue, list,stacks, sets, classes, streams, ranges.
