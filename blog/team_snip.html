What is Snip?

"Your code ... Snipped !"

Snip is a solution to the mechanics of studying code.

During phase 0 and phase 1, it becomes clear that we students type pages upon pages of code. And unlike English, code of a programming language and its function are NOT usually self explanatory. The moment the code leave our fingers and make themself comfy onscreen, they blend in and become camouflaged.

Let's say, after 2 hrs of typing 10 scrolls of code, you had an orgasmic AHA moment where you are f*ing LOVING your code here, here and... here. So of course, you high five yourself by taking a much needed break to release your neglected bladder. When you come back.... all the lines of code look the same... What the @~!#!~! Which lines of code gave you that AHA feel?!

If only Snip existed before you took that bathroom break. A current solution is to go through each line and rediscover your AHA. (boring) Then you think: wouldn't it be thrifty if I CREATED A CHEATSHEET for assessments day with these AHA's? But that would mean parsing through days, weeks (or months... last months AHA's... dear lordy...) of code and then copying and pasting it to a new page... I'm getting... sleepy... just thinking about it.

Snip can snip that time & effort logarithmically. haHA! (Prove it you say? =D Next Friday, 6pm, you know where.)

How Does Snip Work?

When you have an AHA moment or your code is simply beautiful, you'll have the feeling that you should save it. When you get that feeling, type #< snip> before the code and #< /snip> after the code. Like this:

#< snip>
*code that you love*
#< /snip>

Then at the end of the day or whenever, you just run the program on terminal:
ruby [snip program name] [file or batches of file to be snipped]

A file will automatically be created for you and all the little snips that you had previously inserted the "snip" tags on will be pulled into this file in a neat list. Every time you run the program, the new snips will be added to this one file. (Snip will check for duplicate snips for you.) Nifty huh?
